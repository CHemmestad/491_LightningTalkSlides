Program Context

What Is the MySTEMGrowth Survey?

A before-and-after survey that helps undergraduate researchers reflect on growth throughout their program experience.

  • Captures change across Civic Engagement, STEM Interests, Self-Efficacy, and Outcome Expectations.
  • Students compare pre/post results to understand how the program shaped their trajectory.
  • Provides actionable feedback to participants and program leadership.
Prototyping Questions

Guiding the Prototype

  • Goal: Clarify the design and align it to client expectations through an iterative prototype.
  • Form: A functional skeleton website that isolates subsystems while showcasing core flows.

Prototyping Purposes (cont.)

Exploration & Active Learning

Adopted Strategy

Isolating Subsystems

  • What: Standalone prototypes of critical features and user journeys.
  • Why: Focus effort on specific components we inherited or need to improve.
  • How: Build a demo UI, diagram flows, and validate user interactions before full integration.

Security Prototyping

Protecting Students & Data

Our stack layers modern defenses across authentication, storage, and operational processes.

Bcrypt + Salt
Hashed credentials with rotation-ready configuration for quick security response.
Role-Aware APIs
JWT-based middleware scopes routes per persona and enforces least-privilege access.
Encrypted Stores
At-rest encryption for survey submissions plus audited object storage for exports.
  • Credential hygiene: Bcrypt (12+ rounds) with unique salt per user and planned refresh schedules.
  • Session posture: Short-lived JWTs, sliding refresh tokens, and centralized logout hooks.
  • Data governance: PII tokenization in transit; audit logging for downloads & role escalations.
  • Operational safeguards: Infrastructure-as-code for secrets rotation and environment parity.
Mobile Layout Prototyping

Responsive App

Ensuring every MySTEMGrowth flow—from students to admins—stays intuitive on tablets and phones.

  • Navigation patterns: Floating bottom bars for coordinators; tab bars guide students through multi-step surveys.

Key Takeaways

  • DigitalOcean leads with 5/5 in cost & setup plus solid marks across the board.
  • AWS excels in integration and security but carries higher cost and setup overhead.
  • Heroku/Vercel remain attractive for rapid prototypes yet trail in long-term integration control.
  • GCP and Azure provide enterprise scalability with slightly higher operational overhead.
